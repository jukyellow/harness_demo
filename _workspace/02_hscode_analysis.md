# HS코드 도메인 분석 — 온톨로지 적용 대상 모델

> 작성: hscode-analyst | 산출 스킬: hscode-analysis | 작성일: 2026-05-27
> 목적: 팔란티어 3-layer 온톨로지(Object/Link/Action) 적용 대상으로서 HS코드 도메인의 구조·규칙·엔티티·프로세스를 정리한다.

---

## 1. HS코드 개요 및 국제적 위상

**HS(Harmonized Commodity Description and Coding System, 통일상품명 및 부호체계)** 는 세계관세기구(WCO, World Customs Organization)가 제정·관리하는 국제 표준 상품분류 체계다. 무역거래 대상 상품을 체계적으로 분류하기 위한 다목적 코드로, 다음 용도로 전 세계 200여 개국·경제권에서 사용된다.

- **관세 부과**: 수출입 물품에 관세율을 매기는 기준
- **무역통계**: 국가 간 비교 가능한 무역 통계 산출
- **원산지·FTA**: 협정세율·원산지 판정의 기준 코드
- **규제·검역**: 수입요건, 검역, 환경규제(MEA 대상품목) 식별

**핵심 특성**
- WCO가 정의하는 **국제 공통 부분은 앞 6자리**(2·4·6자리)이며, **7자리 이후는 각국이 6자리 범위 내에서 자체 세분**한다.
- 5~6년 주기로 개정되며 현행은 **HS 2022판**(2022.1.1 발효). HS 2022는 351개 개정 세트, 환경·신기술 품목(전자폐기물, 드론, 스마트폰 등) 신설을 포함한다.
- 한국은 이를 확장한 **HSK(HS of Korea), 10자리 관세·통계통합품목분류표**를 운용한다.

> 인용(관세청): "HS는 무역거래의 대상이 될 수 있는 상품을 21부(Section)로 대분류하고, 이를 다시 97류(Chapter)로 중분류(제77류는 유보)한 다음, 호(Heading)로 소분류 및 소호(Sub-heading)로 세분하고 있습니다."

## 2. 계층 구조 (부/류/호/소호/HSK)

HS코드는 상위에서 하위로 내려갈수록 품목이 구체화되는 **엄격한 트리(계층) 구조**다. 각 단계는 자릿수와 식별 단위가 명확히 정의된다.

| 계층 | 자릿수 | 단위명 | 개수(HS 2022 / 한국 기준) | 의미 |
|------|--------|--------|--------------------------|------|
| 부 | (코드 없음) | Section | 21개 | 산업·상품군 대분류 (예: 제16부 기계·전기기기) |
| 류 | 1~2자리 | Chapter | 96개 운영(번호 1~97, **제77류 유보**) | 품목 군 (예: 제85류 전기기기) |
| 호 | 3~4자리 | Heading | 1,228개 | 품목 (예: 8517 전화기) |
| 소호 | 5~6자리 | Subheading | 5,612개 (**WCO 국제표준 최하위**) | 세부 품목 |
| (한국) HSK | 7~10자리 | (통계세분) | 약 12,000여 개 | 한국 통계·관세 세분류 |

**자릿수 해석 규칙**
- 1~2자리 = 류(Chapter)
- 3~4자리 = 호(Heading)
- 5~6자리 = 소호(Sub-heading) — **여기까지 국제 공통**
- 7~10자리 = 한국 고유 세분 (HSK)

> 인용(관세청): "HS 코드 6자리까지는 국제적으로 공통으로 사용하는 코드이며, 7자리부터는 각 나라에서 6단위 범위 내에서 이를 세분하여 10자리까지 사용할 수 있으며, 우리나라는 마지막 4자리를 세분하여 10단위로 사용하고 있습니다."

**HS vs HSK 분리 (중요)**
- **HS 6자리**: WCO 국제표준. 전 세계 동일. 변경 불가(WCO 개정으로만 변경).
- **HSK 10자리**: 한국 관세청이 6자리 아래에 4자리를 추가한 국내 확장. 국가별 상이.
- 온톨로지 설계 시 이 둘은 **동일 객체의 다른 레벨(level)** 로 보되, "국제표준 여부(isInternational)" 속성으로 구분해야 한다.

**부속 규범(분류 판단의 법적 근거)**
- **부 주(Section Notes)** / **류 주(Chapter Notes)**: 각 부·류에 적용되는 정의·포함·제외 규정. 분류 결정 시 호의 용어와 함께 **법적 효력**을 가진다(통칙 1).
- **호 해설서(Explanatory Notes, HSEN)**: WCO 발간 공식 해설. 법적 구속력은 없으나 분류 해석의 표준 참고자료.

## 3. 분류 규칙 — HS 통칙(GRI, General Rules for the Interpretation)

GRI는 상품을 HS 어느 코드에 분류할지를 결정하는 **6개의 일반 해석 규칙**이며, **반드시 순서대로(1→6) 적용**한다. 이는 단순 참조 문서가 아니라 **실제 의사결정 절차(알고리즘)** 이므로, 온톨로지 **Action Layer 매핑의 핵심 입력**이다.

| 통칙 | 명칭 | 핵심 내용 | 적용 레벨 |
|------|------|----------|-----------|
| **GRI 1** | 호의 용어·부/류 주 우선 | 분류는 **호(4자리)의 용어**와 관련 **부 주·류 주**에 따라 우선 결정한다. 다른 모든 규칙에 우선한다. | 호(4자리) |
| **GRI 2(a)** | 미완성·미조립품 | 미완성·미조립(분해) 물품이 완성품의 **본질적 특성(essential character)** 을 갖추면 완성품으로 분류한다. | 호 |
| **GRI 2(b)** | 혼합물·복합물 | 재료·물질의 혼합·결합물은 GRI 3 원칙에 따라 분류한다. | 호 |
| **GRI 3(a)** | 최협(最狹) 특정 우선 | 둘 이상의 호에 해당 가능 시 **가장 구체적으로 표현한 호**를 우선한다. | 호 |
| **GRI 3(b)** | 본질적 특성 | 3(a)로 불가 시 **본질적 특성을 부여하는 재료·구성요소**의 호로 분류한다. | 호 |
| **GRI 3(c)** | 최종 호(numerically last) | 3(a)·3(b)로 불가 시 **번호 순서상 가장 마지막 호**로 분류한다. | 호 |
| **GRI 4** | 가장 유사한 물품 | GRI 1~3으로 분류 불가 시 **가장 유사한 물품**의 호로 분류한다. | 호 |
| **GRI 5(a)** | 전용 용기·케이스 | 특정 물품 전용으로 제작된 케이스(악기·카메라 케이스 등)는 내용물과 함께 분류한다. | 호 |
| **GRI 5(b)** | 포장 재료·용기 | 통상의 포장재·용기는 내용물과 함께 분류(반복 사용 명백 시 별도). | 호 |
| **GRI 6** | 소호 레벨 적용 | **소호(6자리)** 분류는 소호의 용어·소호 주에 따르며, GRI 1~5를 **동일 레벨 소호 간에 준용(mutatis mutandis)** 한다. | 소호(6자리) |

**적용 흐름(의사결정 순서)**
```
GRI 1 (호 용어 + 부/류 주) ──결정?── 예 → 호 확정 → GRI 6 (소호 결정)
        │ 아니오
        ▼
GRI 2 (미완성/미조립, 혼합물) → 필요시 GRI 3
        ▼
GRI 3(a) 최협 특정 → 3(b) 본질적 특성 → 3(c) 최종 호
        ▼
GRI 4 (가장 유사) → 호 확정 → GRI 6 (소호 결정)
        ▼
GRI 5 (포장·용기 부수 처리)
```

> 온톨로지 시사점: GRI는 "입력=품목 특성, 출력=HS코드"인 결정 함수다. Action Layer에서 `분류결정(Classify)` 액션의 **전제조건(precondition)·실행순서·분기조건**으로 매핑된다.

## 4. 핵심 엔티티 및 속성

온톨로지 **Object Layer** 후보 객체와 주요 속성. (★ = 핵심 객체)

| 엔티티 | 설명 | 주요 속성 |
|--------|------|-----------|
| ★ **품목(Commodity)** | 분류 대상이 되는 실제 상품 | 명칭, 재질/성분, 용도, 기능, 완성도(완성/미완성), 형태, 본질적특성 |
| ★ **분류항목(HSCodeItem)** | HS/HSK 코드 노드 자체 | 코드값, 레벨(부/류/호/소호/HSK), 자릿수, 품명, 상위코드, 국제표준여부(isInternational), 개정판(HS2022 등) |
| **부 주/류 주(SectionNote/ChapterNote)** | 부·류에 적용되는 법적 정의·포함·제외 규정 | 적용범위, 정의, 포함규정, 제외규정 |
| ★ **분류규칙(GRI)** | 통칙 1~6 | 규칙번호, 적용레벨(호/소호), 적용순서, 분기조건 |
| **호 해설서(ExplanatoryNote)** | WCO 공식 해설(HSEN) | 대상호, 해설본문, 법적구속력=없음 |
| ★ **관세율(TariffRate)** | 코드별 세율 | 코드, 세율종류(기본/WTO양허/FTA협정/잠정), 세율값, 적용기간 |
| **원산지(Origin)** | FTA 원산지 판정 관련 | 원산지국, 적용협정, 원산지결정기준(세번변경/부가가치) |
| ★ **사전심사(AdvanceRuling)** | 품목분류 사전심사 회신 | 신청인, 신청품목, 회신코드, 회신일, 효력기간, 재심사여부 |
| **분류분쟁(ClassificationDispute)** | 이의신청·심사·심판·소송 | 쟁점코드, 불복단계, 처리기관, 결과 |
| **신고(Declaration)** | 수출입 신고 건 | 신고번호, 신고품목, 적용코드, 신고인 |

## 5. 엔티티 간 관계 (Link Layer 후보)

| 출발 엔티티 | 관계(동사) | 도착 엔티티 | 비고 |
|-------------|-----------|-------------|------|
| 품목 | ─[분류된다]→ | 분류항목(HSCodeItem) | 핵심 관계. GRI 적용 결과 |
| 분류항목 | ─[상위에 속한다]→ | 분류항목(상위 레벨) | 부→류→호→소호→HSK 계층(self-link) |
| 분류항목(HSK) | ─[국제표준을 확장한다]→ | 분류항목(6자리 소호) | HSK 10자리 ↔ HS 6자리 |
| 부 주/류 주 | ─[적용된다]→ | 분류항목(부/류) | 통칙 1의 법적 근거 |
| GRI | ─[지배한다]→ | 품목→분류항목 결정 | 분류 결정 절차 |
| 호 해설서 | ─[해설한다]→ | 분류항목(호) | 비구속 참고 |
| 분류항목 | ─[세율을 가진다]→ | 관세율 | 코드별 다중 세율(기본/양허/FTA) |
| 품목 | ─[원산지를 가진다]→ | 원산지 | FTA 적용 |
| 원산지+분류항목 | ─[협정세율을 결정한다]→ | 관세율(FTA) | 원산지×코드 조합 |
| 사전심사 | ─[확정한다]→ | 분류항목 | 법적 효력 있는 코드 회신 |
| 분류분쟁 | ─[다툰다]→ | 분류항목(쟁점) | 이의·심사·소송 |
| 신고 | ─[적용한다]→ | 분류항목 | 실제 통관 적용 |

**관계 그래프 핵심 경로**
`품목 → (GRI 적용) → 분류항목 → 관세율` (+ `원산지` 결합 시 FTA 세율)

## 6. 운영 프로세스 (분류 결정·사전심사·분쟁)

온톨로지 **Action Layer** 후보 프로세스.

### 6.1 신규 품목 분류 결정 흐름
```
1. 품목 정보 수집 (명칭·재질·용도·기능·완성도)
2. GRI 1 적용: 호 용어 + 부 주/류 주 검토 → 후보 호 도출
3. 다수 호 경합 시 GRI 2~4 순차 적용 → 호(4자리) 확정
4. GRI 6 적용: 소호(6자리) 확정
5. 한국 HSK 세분(7~10자리) 결정 → 10자리 코드 확정
6. 코드에 연결된 관세율·수입요건 확인
```

### 6.2 품목분류 사전심사(Advance Ruling) 제도
수출입신고 전 스스로 분류가 어려운 경우, **관세청 관세평가분류원장**에게 신청하면 **법적 효력이 있는 품목번호**를 결정·회신하는 민원회신 제도.

```
신청(인터넷 portal.customs.go.kr / 우편 / 방문, 견본·증빙 첨부)
   ↓
관세평가분류원 품목분류과 접수 (처리기한 30일, 보정·분석 기간 제외 / 신속처리 15일)
   ↓
품목번호 결정·회신 (법적 효력 발생)
   ↓
[불복 시] 통지일로부터 30일 이내 1회 한정 재심사 요청
```
- **신청 제한**: 범칙조사·불복(과세전적부심·심사청구·심판청구) 진행 중인 경우 신청 제한.

### 6.3 분류 분쟁(불복) 처리 흐름
```
처분(분류·과세) → 이의신청 / 과세전적부심사
   ↓
심사청구(관세청) 또는 심판청구(조세심판원)  ← 행정 단계
   ↓
행정소송(법원)  ← 사법 단계
```
- 사전심사 단계의 **재심사 요청(분류원 내부)** 과 처분 후의 **불복 절차(이의·심사·심판·소송)** 는 구분된다.

## 7. 온톨로지 적용을 위한 도메인 모델 요약 (architect 인계용)

| 팔란티어 레이어 | HS코드 도메인 매핑 후보 |
|-----------------|------------------------|
| **Object** | 품목, 분류항목(HSCodeItem), 관세율, 원산지, 사전심사, 분류분쟁, 부주/류주, 호해설서, 신고 |
| **Link** | 품목→분류된다→분류항목, 분류항목→상위계층, HSK→확장→소호, 분류항목→세율, 사전심사→확정→분류항목, 분류분쟁→다툰다→분류항목 |
| **Action** | GRI 1~6 분류결정 절차, 사전심사 신청·회신·재심사, 불복(이의·심사·심판·소송), HSK 세분 결정 |

**핵심 설계 유의점**
1. **HS(6자리 국제표준) vs HSK(10자리 한국 확장)** 은 동일 객체의 레벨 차이로 모델링하되 `isInternational` 속성으로 분리.
2. **계층은 self-referential link**(부→류→호→소호→HSK)로 표현.
3. **GRI는 단순 참조가 아니라 Action(결정 절차)** — 전제조건·실행순서·분기를 명시.
4. **부 주/류 주는 법적 효력을 갖는 분류 근거** — 단순 메타데이터가 아닌 Object+Link로 모델링.
