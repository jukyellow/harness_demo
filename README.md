
# HS 온톨로지 실험실

1. HS 온톨로지 구축 리서치
 - 기술/방법론/도메인지식/적용사례 리서치
 - MVP 구축을 위한 기획안 검토/작성
 - 아래 분석/설계/MVP구축 가이드라인 평가/개선
  
2. HS 온톨로지 분석
 - 관세법령정보포털>관세율표>한국2026>HS2~10단위 상세페이지 분석(수집할정보, 데이터 realtion 파악 등)
 - 수집할 데이터 범위/수집방안/활용방안 검토
 - 구축때 활용할 엔진/기술 선정
 - 전체 수집/서빙/추론 파이프라인 검토
 - 분석 산출물 작성

3. HS 온톨로지 요구사항 정의
 - 분석결과를 바탕으로 요구사항 정의
 - claude가 요구사항을 도출하고 선택할수 있는 경우의수(후보군)를 제시하면 사용자가 결정
 - 본 검토문서에 이미 많은 요구사항들이 도출되어있는데, 이걸 체계적으로 정리필요
 - 산출물(요구사항정의서 등) 작성
 - 정리된 요구사항을 바탕으로 WBS작성(작업범위/일정/역학별수행내용/완료여부 체크) 및 서브Agent들이 개별적으로 수행하기위한 작업지시서(작업할내용,검수방법, 완료Gate등) 작성
  
4. HS 온톨로지 설계
 - 3-layer 구조 설계
 - 수집할 데이터 구조 정의
 - 데이터간 relation 정의
 - 3계층간 역할/활용/구축방법 정의
 - Agentic HS 추론을 위한 Agentic flow 설계
   - 데모 시나리오: 사용자 특정HS2류 내에서 질문->HS 2~10단위 구분을 위한 판단(각 단위에서 후보군 도출, 추가로 필요한 정보요청, HS10단위까지 cycle반복)->최종 추천한 HS10단위에는 각 단위를 분류한 근거제시 필요 
 - 데모를 위한 서빙서버 및 화면설계
 - HS분류 로깅/시각화 설계(관리자용)
   - HS분류 과정을 상세히 로깅을 남기고 해당 로그를 기반으로 시각화 구성(사용자 질문과 HS2~10단위 판별의 근거/증적/확률 등을 쉽게 파악할수 있게)   
 - 각 파이프라인 단계별 검수Gate 선정 및 검수방안 설계
 - 평가 방법론 설계
 - 기타 고려사항 검토
 - 설계산출물 검토
  
5. HS 온톨로지 MVP 구축
 - 설계를 바탕으로 특정 HS2단위 선정
 - 데이터 수집 > 정제 > 온톨로지 구축 > 서빙서버구축 > 데모화면 구축(사용자,관리자) 순서로 진행
 - 각 단계별 검수진행
 - 데모화면에서 사용자 데모 실행가능하도록 MVP 완성
 - MVP 구축 산출물작성

6. 최종보고서 작성
 - 작업과정 요약/정리
 - 성과/산출물 설명
 - 작업의 의의
 - 향후 전체(HS97류) 확장시 고려사항 등 정리
